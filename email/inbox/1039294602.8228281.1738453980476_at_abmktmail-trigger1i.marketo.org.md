Thanks for starting a 90-day trial of GraphOS!
The GraphOS trial gives you access to a toolkit for building, running, and optimizing a GraphQL API across teams of any size. 
If you’ve already got a graph running on either Apollo Router or Apollo Server, you’re off to a great start! Just connect your graph <https://em.apollographql.com/MDU4LUpGVS04NTEAAAGYYudFMJXlOXJHDA4QYASJB7tSbrcDlSU49xKURkZLr8A7UKhm4yQGi5FQa4znwvttoYEOhp0=> to your new GraphOS organization to get access to more advanced features and to start reporting metrics and schemas to GraphOS Studio.
If you don’t have a graph yet, the fastest way to get started is with our quickstart guide <https://em.apollographql.com/MDU4LUpGVS04NTEAAAGYYudFMCGjLYC2FnV0tEO4hSsG1tdLjPuuwdWswW3lF0QLufNymb6L8jqTRQu-BL6aZpnN8xI=> which provides demo subgraphs and schemas for you to use and walks you through setting up a self-hosted instance of Apollo Router.
Once you get set up and authenticated, you'll get to try out Apollo’s enterprise-grade capabilities like Explorer, graph variants (for different environments or use cases), checks (to validate schema changes against historical operations), and more. You’ll also get to try out enterprise capabilities of Apollo Router like JWT auth, operation safety limits, and query safelisting. 
That’s all for now! If you have any questions or run into any trouble getting started, don’t hesitate to reach out to our team <https://em.apollographql.com/MDU4LUpGVS04NTEAAAGYYudFMPsHCUg0YWfhYBxpxv8RRaMd-08nEplEr857HTuxhp2MkLdbi5P0ohaI0OEmqmXDRMg=> or join our community in the Apollo Discord <https://em.apollographql.com/MDU4LUpGVS04NTEAAAGYYudFMMYkbYQVC0odLdGwDliXfgKR0kxs1FMxDyRDFOjSgcc-Cgywp9gY-WB3YaK7f_MfPNY=> (you’ll even get access to a private POC channel where we can help you out).
