Here are some basic best practices for your new ClickHouse service:

Use bulk or async inserts

ClickHouse performs best when inserts are batched <https://talk.clickhouse.com/dc/6JTfYqzk_peg2k6s9VYOscTzVvLVEeNT2G45in3kqdUMj7hLHaMW3vi2GGzDZQFfLxwRrUX8k5wyoC7NlgOpPJppUzPnaN1XYAFfXWAFAgDfUTW88uUUB_H95UH5lA4fkNeqclN2zNMaTFFG5TU_iTKNGcYgpLPkTbErFPdhTQOsxjG4qQL57cSc0zTPycQq/MjM4LUZQQy0zMTcAAAGYuIwqPb6L8atxLi4hSJUXivMMoIRH_twgHK23ceu56Y4k3DFzYc8VXX9Ku0kyaLVoXLBS8ps=> (for example, for rows of 1000) as opposed to many rows individually, especially for large datasets. If you prefer not to batch inserts, we recommend using the async insert option. When using async inserts <https://talk.clickhouse.com/dc/6JTfYqzk_peg2k6s9VYOscTzVvLVEeNT2G45in3kqdXyceeUVjFeNk1OWKIj5VxVQ5r7aWVimgAPkxySgmJykc4Je_VSa6IMsngkyW4SEs3PSaUB03n7bq8jUyJgbEiPypzgSsqxM9RcXr07U60g3RraiCfPXxtYTsygIXi4W4yNmf01msc9JU9zIzgTWEYcltlfUF6fbUcsFoNSBFvD_w==/MjM4LUZQQy0zMTcAAAGYuIwqPb6L8atxLi4hSJUXivMMoIRH_twgHK23ceu56Y4k3DFzYc8VXX9Ku0kyaLVoXLBS8ps=>, ClickHouse will batch inserts for you.

Limit usage of mutations

Mutations are commands that alter data, such as UPDATE or DELETE. While we recently released the lightweight update and delete operations <https://talk.clickhouse.com/dc/6JTfYqzk_peg2k6s9VYOsdHA4FfnyKRpRHwAXRDB4yvS8hyKetil5AcmPl5zZdjG5PRBWXdnP7ccCQ9YTEOcp1slPSR2ORRBLvRr_UxVfLhwT-htnEBijO3E3FBBT8LcQE_ClUPhIOAfXxDTvjTOHnY3N3zUaZ0uqSnEJtmCdtQOeg5JVNlMqIuOzS4iAR-DgjH54DmVIxJhW2AvfOJ32g==/MjM4LUZQQy0zMTcAAAGYuIwqPb6L8atxLi4hSJUXivMMoIRH_twgHK23ceu56Y4k3DFzYc8VXX9Ku0kyaLVoXLBS8ps=>, we still recommend limiting mutations for the best performance. It is best to design your schema and query access patterns in such a way that you edit data in ClickHouse only when necessary <https://talk.clickhouse.com/dc/6JTfYqzk_peg2k6s9VYOscTzVvLVEeNT2G45in3kqdWwsQM2cZt7XcQ_h5G-xYMcjyZWHTVDOMTduMcKAw-mBjH4MVwcY1NhdgilRrS7lHmuMhTqE02Gnzp7kA6EuhD54L4b6-l3UOHfIP0ni40rsvl3atbTThRHck-rg98i6y4zva25StSNdMVt1diyrPK8/MjM4LUZQQy0zMTcAAAGYuIwqPb6L8atxLi4hSJUXivMMoIRH_twgHK23ceu56Y4k3DFzYc8VXX9Ku0kyaLVoXLBS8ps=>.

Avoid nullable columns

Columns that can be null take extra space and will require additional memory for queries. Instead of making columns nullable, we recommend setting a default value for columns, such as 0 for Int columns and an empty string for String columns.
