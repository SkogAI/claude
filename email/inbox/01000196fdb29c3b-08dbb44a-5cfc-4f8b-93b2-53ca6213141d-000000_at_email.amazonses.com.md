🔭 OTel-based Python SDK (Beta)

Starting today, you can try the next generation of the Langfuse Python SDK, now in public beta and fully rebuilt on
the OpenTelemetry standard.
This is a big upgrade focused on improving developer experience and making it easier than ever to trace your LLM applications.
With native OTel support, context propagation is automatic and third-party libraries instrumented with OpenTelemetry seamlessly
integrate into your Langfuse traces.
Why you’ll love it:
 * Unified context propagation: Automatically nests spans and generations, even across threads or async tasks.

 * Simple & Flexible Instrumentation: Use the @observe decorator, context managers, or manual observations. Mix and match as you
   like.

 * Ecosystem Integration: Works out-of-the-box with OTel-instrumented libraries like MLflow, OpenLit, and more. This is especially
   interesting as it covers the long-tail of model and agent SDKs much better.

 * Global Client Access: Easily fetch the initialized Langfuse client anywhere in your project via get_client().

 * 🕹️ Get started now: Install the beta with pip install langfuse>=3.0.0b1

→ Find out more in the changelog [https://langfuse.com/changelog/2025-05-23-otel-based-python-sdk] & docs
[https://langfuse.com/docs/sdk/python/sdk-v3]
