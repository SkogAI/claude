Introducing WebCode
*******************

В 
Today we announced ( https://x.com/ExaAILabs/status/2036136863682171058 ) we're open-sourcing WebCode, the eval suite we built to assess web search capabilities for coding agents.

Software engineering is the primary use case for AI agents by volume, yet there are major gaps in how existing evals measure code search retrieval and content quality.

--------------------
Why we built WebCode
--------------------

В 
At Exa, we've observed a growing trend in code search queries over the past year, and yet have found limitations in the benchmarks that are currently available:

1. Public coding benchmarks are saturated and contaminated. OpenAI recently deprecated SWE-bench Verified because models that have seen the problems during training succeed via memorization, not reasoning.

2. Correctness-only metrics are inherently flawed. Our research identified cases where LLMs answer correctly
despite incorrect web search data because it had memorized the docs from training data.

3. Evaluation should be a discriminative task: does the retrieved context actually contain the answer? This cleanly separates search quality from model quality.
