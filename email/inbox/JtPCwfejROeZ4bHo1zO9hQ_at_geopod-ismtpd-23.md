Get secret protection, code security, or both!
----------------------------------------------

Starting April 1, 2025, GitHub Advanced Security (GHAS) will be unbundled ( https://github.blog/changelog/2025-03-04-introducing-github-secret-protection-and-github-code-security/?utm_campaign=03-11-2025-Product-Update-Email&utm_medium=email&utm_source=copy-link ) and offered as two standalone security products:

* GitHub Secret Protection (previously GitHub secret scanning)
* GitHub Code Security (previously GitHub code scanning)

This change will enable you to pick only the security features that you need, making enterprise-grade security more affordable. Also, GitHub Team plan customers will be able to purchase these services without the need for a GitHub Enterprise subscription.

Dependabot alerts now include exploit likelihood data (EPSS)
------------------------------------------------------------

Available on GitHub.com and GHES 3.17, Dependabot now features exploitability data ( https://github.blog/changelog/2025-02-19-dependabot-helps-users-focus-on-the-most-important-alerts-by-including-epss-scores-that-indicate-likelihood-of-exploitation-now-generally-available/?utm_campaign=03-11-2025-Product-Update-Email&utm_medium=email&utm_source=copy-link ) to help you focus on the 10% of alerts that matter most.

Vulnerability alerts now include Exploit Prediction Scoring System (EPSS) data to help you prioritize alerts based on exploit likelihood. Only 10% of vulnerability alerts have an EPSS score above 0.95%. This makes EPSS a powerful tool for prioritization, especially when used in conjunction with exploitation severity (CVSS).
