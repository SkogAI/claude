Here’s a roundup of what shipped in the last month, including new enterprise tools for managing Copilot usage, a collection of models and IDE improvements, as well as updates around connecting to MCP servers.

( https://github.blog/changelog/2025-11-03-control-ai-spending-with-budget-tracking-for-github-ai-tools/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec )

Giving you greater control over your enterprise’s use of GitHub Copilot
-----------------------------------------------------------------------

* Have more fine-grained control over your Copilot budgets: ( https://github.blog/changelog/2025-11-03-control-ai-spending-with-budget-tracking-for-github-ai-tools/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Use the new SKUs specifically for GitHub Copilot coding agent and GitHub Spark to control premium usage just for those tools.
* Programmatically manage your Copilot budget: ( https://github.blog/changelog/2025-11-03-manage-budgets-and-track-usage-with-new-billing-api-updates/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Use APIs to control your budgets and review AI usage.
* Delegate AI management: ( https://github.blog/changelog/2025-11-03-delegate-ai-controls-management-to-members-of-your-enterprise/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Grant enterprise members access to your agent control plane ( https://github.blog/changelog/2025-10-28-enterprise-ai-controls-the-agent-control-plane-are-in-public-preview/?utm_campaign=mpu_Dec&utm_medium=email&utm_source=github ) by giving them a new role ( https://github.blog/changelog/2025-11-17-fine-grain-permissions-for-copilot-usage-metrics-now-available/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) created for this purpose.
* Customize Copilot to your needs: ( https://github.blog/changelog/2025-11-05-copilot-coding-agent-supports-organization-custom-instructions/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Custom instructions allow you to guide Copilot to align with your own coding standards and practices.

( https://github.blog/changelog/2025-11-18-gemini-3-pro-is-in-public-preview-for-github-copilot/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec )

Adding models and enhancing IDEs
--------------------------------

* *Claude Haiku 4.5 ( https://github.blog/changelog/2025-11-10-claude-sonnet-3-5-deprecated-claude-haiku-4-5-available-in-copilot-free/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) , Claude Opus 4.5 ( https://github.blog/changelog/2025-11-24-claude-opus-4-5-is-in-public-preview-for-github-copilot/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) , multiple OpenAI GPT 5.1 models ( https://github.blog/changelog/2025-11-13-openais-gpt-5-1-gpt-5-1-codex-and-gpt-5-1-codex-mini-are-now-in-public-preview-for-github-copilot/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) , and Gemini 3 Pro are now available:* You can select these latest models across the suite of Copilot experiences.
* *Additional features for JetBrains, Xcode, and Eclipse:* Take advantage of plan mode ( https://github.blog/changelog/2025-11-18-plan-mode-in-github-copilot-now-in-public-preview-in-jetbrains-eclipse-and-xcode/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) and auto model selection ( https://github.blog/changelog/2025-11-18-auto-model-selection-for-copilot-in-jetbrains-ides-xcode-and-eclipse-in-public-preview/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) in each of these IDEs, and next edit suggestions (NES) ( https://github.blog/changelog/2025-11-18-github-copilot-next-edit-suggestions-nes-now-in-public-preview-for-xcode-and-eclipse/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) when using Xcode or Eclipse.
* Update to the latest version of VS Code: ( https://code.visualstudio.com/updates/v1_106?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Review the latest release notes for the current version of VS Code.

( https://github.blog/changelog/2025-11-18-internal-mcp-registry-and-allowlist-controls-for-vs-code-stable-in-public-preview/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec )

A focus on security
-------------------

* Configure an MCP registry and limit MCP access: ( https://github.blog/changelog/2025-11-18-internal-mcp-registry-and-allowlist-controls-for-vs-code-stable-in-public-preview/?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Create an MCP registry in VS Code Stable and use it to aid in MCP server discoverability, restrict access to listed MCP servers, or both.
* GitHub Advanced Security integrates with Microsoft Defender for Cloud: ( https://techcommunity.microsoft.com/blog/microsoftdefendercloudblog/microsoft-defender-for-cloud-innovations-at-ignite-2025/4469386?utm_source=github&utm_medium=email&utm_campaign=mpu_Dec ) Use Microsoft’s Defender for Cloud to boost visibility between security and development teams.
