Load balancing endpoints in public beta: direct HTTP access to your workers ( https://console.runpod.io/serverless/new-endpoint )
*********************************************************************************************************************************

Our new load balancing endpoints route traffic directly to your
workers' web servers, unlocking real-time streaming, custom routes, and sub-second latency for
interactive applications.

Run vLLM at full throttle, expose multiple endpoints from a single
worker, or build complex microservices with FastAPI or Flask, all while maintaining the
scalability and reliability of Runpod's infrastructure.

Just implement a
/ping health check, deploy your HTTP server, and watch as requests flow directly to
healthy workers through custom paths like
https://ENDPOINT_ID.api.runpod.ai/YOUR_CUSTOM_PATH.

Check out our docs ( https://docs.runpod.io/serverless/load-balancing/overview ) to learn more about load balancing endpoints.
