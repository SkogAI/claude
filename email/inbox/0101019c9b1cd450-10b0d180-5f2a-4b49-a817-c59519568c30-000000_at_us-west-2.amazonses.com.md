Here's what's new in Lovable for your team:

SCIM provisioning

Automatically provision and deprovision users from your identity provider. Map IdP groups to Lovable roles (Viewer, Editor, Admin) so new users get the right permissions from day one. Available on Enterprise plans. Learn more ( https://docs.lovable.dev/features/business/scim )

Cross-project referencing

The agent can now access code, components, and features from other projects in your workspace. Reference a project in chat, and Lovable reads how it was built, then recreates or adapts it in your current project. The original stays untouched. Projects can opt out in settings, and admins can disable the feature workspace-wide. Learn more ( https://docs.lovable.dev/features/cross-project-referencing )

Cross-project referencing in Lovable

Admin controls for shared connectors

Workspace admins and owners on Business and Enterprise plans can now enable or disable specific shared connectors at the workspace level. Lovable Cloud cannot be disabled. This gives teams control over which third-party services are available across the workspace. Learn more ( https://docs.lovable.dev/integrations/introduction#shared-connectors-add-capabilities-to-your-published-app )

Personal folders

Folders can now be set to Workspace (visible to all members) or Private (visible only to the creator). Projects inside inherit the folder's visibility. When moving a project, you get a preview of who gains or loses access.

Slack, Granola, and Amplitude connectors

Build Slack ( https://docs.lovable.dev/integrations/slack )-powered notifications and workflows into your apps. Connect Granola to turn meeting notes into working prototypes, and Amplitude to build with your product analytics. Learn more ( https://docs.lovable.dev/integrations/mcp-servers )

Cloud region selection

You can now choose where your Cloud project is hosted: Americas, Europe, or Asia Pacific. If you set a preferred default region in Cloud connector settings, Lovable uses that region automatically. Choosing a nearby region improves latency and overall backend performance. Learn more ( https://docs.lovable.dev/integrations/cloud )

Also improved

External collaborator warnings when inviting someone outside your organization. File edits now persist when you stop the agent. Faster project search with full-text matching.
