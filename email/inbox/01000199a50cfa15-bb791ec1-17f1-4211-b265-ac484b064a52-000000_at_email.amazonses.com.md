Hi Emil Skogsund,
September was a big month. We shipped a new Experiment Runner SDK, introduced our first AI feature, rolled out structured outputs
for prompt experiments, graduated our TypeScript SDK v4 to GA, and a 50% price drop for the Core plan.
Quick highlights below. Links go straight to docs so you can dive in fast.
 


Experiment Runner SDK

https://langfuse.com/docs/evaluation/experiments/experiments-via-sdk#experiment-runner-sdk

We've added a new high-level SDK abstraction that makes running experiments on datasets a breeze. It comes with automatic tracing,
concurrent execution, and flexible evaluation built in.
This is now the recommended way to run experiments on local or Langfuse-hosted datasets with our Python and JS/TS SDKs.
→ Learn more in the docs [https://langfuse.com/docs/evaluation/experiments/experiments-via-sdk#experiment-runner-sdk]
 


Natural Language Filtering for Traces

https://langfuse.com/changelog/2025-09-30-natural-language-filters

You can now filter your traces and observations using plain English. Just describe what you're looking for, and Langfuse will
construct the right filters for you.
You can then refine them manually if needed. This is our first AI feature in Langfuse, with more to follow!
→ See how it works [https://langfuse.com/changelog/2025-09-30-natural-language-filters]
 


Structured Output Support for Prompt Experiments

https://langfuse.com/changelog/2025-09-30-structured-output-experiments

Ensure your LLM responses conform to a specific JSON schema in your Prompt Experiments. Enforcing a structured output makes it
much easier to evaluate outputs consistently and process results programmatically.
→ Read the changelog [https://langfuse.com/changelog/2025-09-30-structured-output-experiments]
 


TypeScript SDK v4 (GA)

https://www.youtube.com/watch?v=BjEXs13KyV4

We’ve taken JS/TypeScript tracing up a notch: The Langfuse JS/TypeScript SDK v4
[https://langfuse.com/docs/observability/sdk/typescript/overview] is out of beta as of September 2025!
We’ve rebuilt this SDK on top of OpenTelemetry to improve the developer experience, make context management more robust, and allow
for easy integrations with the JS/TS ecosystem (Provider SDKs, Vercel AI SDK, LangChain JS, Mastra, ...).
→ See the documentation [https://langfuse.com/docs/observability/sdk/typescript/overview]
→ Upgrade now from v3 to v4 [https://langfuse.com/docs/observability/sdk/typescript/upgrade-path]
 
If you are on Python, all of this is already available in the current version of the Python SDK.


More September Releases

 * Mutable Score Configs [https://langfuse.com/changelog/2025-09-29-score-config-updates]: You can now edit score configurations
   in Langfuse after they've been created.

 * Core Plan now $29: [https://langfuse.com/pricing] We've dropped the price of our Core plan by 50% from $59 to $29/month.

 * Self-Serve Enterprise Cloud: [https://langfuse.com/pricing] We've introduced a new self-serve Enterprise plan for teams that
   need advanced security, support, and scale.

 


User Highlights

https://langfuse.com/customers/khan-academy

Khan Academy uses Langfuse's public API to power Khanmigo AI, building a custom Golang client and linking internal tools directly
to traces for debugging and analysis.
Since rolling out in April 2024, adoption has expanded to 100+ users across product and infrastructure teams, enabling rapid
iteration, shared visibility, and leadership insights without hosting their own tracing infrastructure.
→ Read the whole story [https://langfuse.com/customers/khan-academy]
