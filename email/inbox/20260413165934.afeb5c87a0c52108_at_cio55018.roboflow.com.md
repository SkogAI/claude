A great computer vision model doesn’t require a huge dataset, just the right steps. Here are five common mistakes that hold models back, and how to avoid them. Here's our blog post about it too ( https://blog.roboflow.com/how-to-improve-your-computer-vision-model/?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ).

1. Picking the wrong model type

Each project type ( https://docs.roboflow.com/datasets/create-a-project?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) in Roboflow corresponds to a model type. We advise picking the simplest model type that will solve your problem. Simpler models like classification ( https://blog.roboflow.com/image-classification/?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) are quick to label and run inference faster, while more complex models like instance segmentation ( https://blog.roboflow.com/instance-segmentation/?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) take more time to label and are slower at inference.
2. Not using data that matches the data your model will see in production

Computer vision models try to match patterns they “see” in your training data. One of the biggest mistakes we see users make is labeling training data that is nothing like the data they expect to see in production. For instance, if you plan on detecting animals on images taken from mobile phones, don’t use Google Image search results for “cat and dog” as training data.

3. Starting with too much data

It’s better to start with a small training dataset (~50-100 images) and focus on ensuring your data is varied. If you’re labeling frames from a video, it’s best to take one image per 5 seconds (or more) and sample from multiple videos. Smaller datasets let you see training results faster, and are faster to relabel later on.

4. Waiting too long to train

Many users wait too long before they train their first model, though it’s better to train early and train often. You can use Roboflow tools like model evaluation ( https://blog.roboflow.com/evaluate-roboflow-models/?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) to see where your model is weak, which helps prioritize future work.
Good news, you’ll soon be able to fine-tune Rapid models. Join the waitlist ( https://rapid.roboflow.com/?open=waitlist&for=continuous-improvement&utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) to try it first.

5. Obsessing over hyperparameters instead of data

Tweaking hyperparameters and adding lots of augmentations can give a strong model a couple extra points of accuracy, but it will not make a bad model good. If you are looking to improve model mAP ( https://blog.roboflow.com/mean-average-precision/?utm_campaign=Onboarding+Campaign+%28Gust+-+02%2F2026%29&utm_content=05+common+mistakes&utm_medium=email_action&utm_source=email ) by 10-20%, focus on
the data. You likely have annotation errors (is every image labeled exactly how you want the model to respond?) or need more varied training data.
