## Gateway API core support

After adding preliminary support last year, the ngrok Kubernetes Operator (https://d2v8tf04.na1.hubspotlinks.com/Ctc/5F+113/d2v8tf04/VWjJX58nwLWHW3PHWpY5ZG_tNVtG5yr5w0zHnN7nzPw43pyd0W7lCdLW6lZ3p_W1sGSN63QwZzxW4x17ny2Gx_PBW3SZ_rs1-kJkXW1cDMj94Mc-1SN4fB3RNqj7Z3W2H816m1qp862W1S8hDj5D8nxGV7Cr-L9fyrVTW8jqCSP58GLkWF4YQWWYNZYBW4ZfgVC6xFhXbN2Mn_PRLn7zqW6Lg2G31kFxyfW32BrrD65kX2zW6lZmk15mBZHfW2LbjcD2c5TpjW12J6wt7Vy9z_N2cypxthtFvKN1jChwFXZtzFN7nk2w3GNLpdMNSYBf9H735W150G-525ZbpXN3Mf3wt5q7gkVXYnn31vD9qxd7MW0j04 ) supports the entire core Gateway API. This includes:

- Routes (HTTPRoute, TCPRoute, TLSRoute) + RouteMatches (header, path, + more)
- Filters: header, redirect, rewrite + more
- Backends: Backend filters + weighted balancing!
- ReferenceGrant: RBAC for multi-tenant clusters handling
- Traffic Policy as an extensionRef or annotation when the Gateway API isn’t flexible enough

In unique ngrok fashion, we compile all of your Gateway API definitions to Traffic Policy and accelerate them by executing those behaviors on our global cloud service.

## --binding=kubernetes

Kubernetes bindings is our zero-config take on Telepresence—these endpoints that are privately accessible only inside of K8s clusters where you’ve installed the ngrok operator. Great for local development and cross-cluster networking between any two production clusters regardless of where they’re running.

```
# on any machine
ngrok http 8080 --url https://d2v8tf04.na1.hubspotlinks.com/Ctc/5F+113/d2v8tf04/VWjJX58nwLWHW3PHWpY5ZG_tNVtG5yr5w0zHnN7nzPvP3pyd0W6N1vHY6lZ3kqW1hXmpw7ySr82VsxW056-L5t4Vh-XFM5qHKQzN3VbS6-5-ZBRW4JzKCG9dtkV1W6Z6PHQ82p5rZW7WKmNR4B1ZvRVDv9jV8mjlQ5VMPfnl7jDMK4W5hkwCx63jChBW5Jn_rG45FyFBVrQ90w2JrNh7V88th54jMhS1W3KT20w6w_B2dW15Zw1k8Mx8p1W927Rxz3PFP36W4VnS0B2VtFFfW4V32Z_29pSWYW1zBxDW99XxzlW2mWkNf5lSHyjW3M_hmd6jv8_7W77y0WS9bbs5mf15cjPR04 --binding kubernetes
```
