**What Triggers Unlock**
========================

Before Triggers, workflows were mostly started by a manual Run or an API call. That works fine for request-response tasks, but gets painful when you need scheduling or event-driven automation.


Triggers move "when to start" onto the canvas itself, so you can configure start conditions, trace runs, standardize inputs, and run workflows as always-on automations with far less overhead.

**Which Trigger should you use?**
=================================

**Schedule Trigger**
--------------------

Run workflows on a set cadence (daily, weekly, monthly). Configure visually or use Cron expressions for precise control. Ideal for reports, data sync, and routine checks.

**Plugin Trigger**
------------------

React to real-time events from tools like Slack, GitHub, Outlook, or Zendesk. The plugin handles auth and event parsing so you can focus on your logic.

**Webhook Trigger**
-------------------

Connect internal systems or custom tools via HTTP. You get full control over method, path, and request/response schemas. For security, we recommend implementing HMAC signature verification in a downstream node.
