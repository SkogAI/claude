Desktop Sandbox for computer use

The new E2B Desktop Sandbox [https://github.com/e2b-dev/desktop] is a secure virtual desktop powered by E2B [https://e2b.dev/] and
made specifically for LLM-powered computer use. Each E2B Sandbox is isolated from the others and can be customized with any
dependencies you want.
Check out also the example [https://github.com/e2b-dev/secure-computer-use] we added.




Built with E2B

 * smolagents [https://huggingface.co/blog/smolagents] by Hugging Face - a library for building coding agents. Enables secure code
   execution via E2B sandboxes

 * Dream [https://askdream.ai/], an agent-building framework & one of the first users implementing E2B sandbox snapshots, allowing
   users to pause and resume immutable sandboxes

 * Jupyter Agents [https://huggingface.co/spaces/data-agents/jupyter-agent] - AI-controlled Jupyter Notebook powered by E2B

 * CAMEL-AI [https://www.camel-ai.org/], a multi-agent framework with E2B as a code interpreter

 * Computer use [https://x.com/jamesmurdza/status/1867025013892866458] with open-source LLMs using E2B as an isolated secure
   environment

 * WebDev Arena [https://web.lmarena.ai/] running comparisons of different LLMs using E2B as the cloud runtime

 * Passive income generator
   [https://github.com/ComposioHQ/composio/tree/master/python/examples/advanced_agents/indie_hacker_agent/langgraph?utm_source=twitter]
   - an AI agent that looks at HackerNews and
   posts, finds and builds the most interesting use case, built with E2B and Composio

 * DemoDrive [https://www.demodrive.tech/] using E2B to run tests of your docs and content

 * Smithery [https://smithery.ai/] adding an E2B MCP server that allows anyone to give code execution to AI agents.
