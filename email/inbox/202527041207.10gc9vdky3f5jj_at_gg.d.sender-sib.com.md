###🚤 TL;DR:

- Connect Dify apps to over 7000 tools using MCP plugins, and share your own Dify apps as MCP servers for other clients to access.

###**↔️ MCP Plugins Enhance Dify Connectivity In and Out**
###=======================================================
 

The Model Context Protocol (MCP) now makes it easier to connect Dify apps both [outwards](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2nfsTSeY62oltNKjy2JTL2Wc/NaqZff9ciDet) and [inwards](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2nfuMSB3P1O0LPh3mCfy7mT8/F9sgM5yaMrWj). Using community plugins like [MCP SSE](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2nfwFRhYhzxEnS3NaN2SuWPe/DO11L21ApenX) and [MCP Agent Strategy](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2nfy8RE40yWTFUPhOXOxhGMA/E5LX5CrTzTwC), you can link your Dify apps to external tools, including the thousands available through services like Zapier MCP. Conversely, you can share your own Dify apps as MCP servers using the [mcp-server](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2ng01QkZJx5hhWm1ChlSU0Ig/WTW_JH7ntnwr) plugin, making their functions available to other MCP clients, which is ideal for internal AI capability sharing (use on private networks is advised). Future native MCP support is also planned.

###** {📄}****Getting More Reliable JSON from LLMs**


 

Getting consistent JSON from LLMs can be tricky. We've added a JSON Schema Editor to the LLM Node to help. Define the structure you need, and the model will follow it, giving you more predictable, reliable outputs for APIs or other workflow steps. [Watch how it works](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2ng3nPnZvuEAbbUep2US3UBi/2GeDIoWC_pry).


###**🧠 Giving Your Workflow Agents Some Memory**


 

Workflow agents can now remember conversation history for better context. Start by updating your agent strategy plugins. Then, simply enable the new "Memory" toggle on the Agent node and adjust the "Window Size" to control the depth of recall for more coherent, context-aware interactions. (Note for custom plugin developers: the `history-messages` feature is available).

Details are in the [Agent Node Docs](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2ng5gPK5EsnP3dqydCqwqE8E/UcrxNk7HS2jP) and [Plugin Dev Docs](https://b6usd.r.ag.d.sendibm3.com/mk/cl/f/sh/7nVU1aA2ng7ZOqaXrMdVgDIRNDRcy4k/u-k1P4KTMbgA).
