Hi,



We’re writing to let you know about an upcoming change to Claude Code on 
Windows that requires action from your IT team.



What’s changing



After March 11, 2026, Claude Code on Windows will no longer read managed 
settings from the legacy path:



Old path (deprecated): C:\ProgramData\ClaudeCode\managed-settings.json

New path: C:\Program Files\ClaudeCode\managed-settings.json



The new path is already supported as of Claude Code 2.0.58.



What you need to do



 Update your endpoint management (MDM) configuration to deploy 
managed-settings.json to the new path before March 11, 2026. No changes to the 
file content are required – only the deployment location.



What happens if you don’t migrate



Starting March 12, 2026, new versions of Claude Code will no longer use the 
old path for managed settings. Any enterprise settings, policies, restrictions, 
and configurations deployed via the old path will no longer be enforced for 
your Windows users.



 If you have questions about this migration, please contact your account 
representative or reach out to support.
