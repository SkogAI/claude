Scalar Docs V2 is rolling out: new navigation + layouts, MDX, and Copy as Markdown. Your WYSIWYG and GitHub Sync projects will move to V2 on deployment.

Scalar Docs V2 is here
New navigation and layout options, MDX support, and LLM-focused workflows (including Copy as Markdown).

Hey Scalar community 👋,

Over the past year, we’ve rebuilt Scalar Docs behind the scenes to support a more flexible documentation experience. Docs V2 brings expanded navigation capabilities, more layout control, and new features designed for modern AI-driven workflows.

What’s new in Docs V2

* More navigation structures and page organisation (nested, grouped, or flattened)
* More layout configurations (including finer control over TOC/sidebar behaviour)
* Faster, more scalable API reference rendering for large OpenAPI specs
* MDX support and richer content composition
* LLM-focused features like *Copy as Markdown*

Automatic upgrade on deployment

*All WYSIWYG and GitHub Sync Docs projects will be automatically upgraded to V2 on deployment.*

* No action required to keep your project live
* Your existing content should continue to work as-is
* Upgrade your config when you’re ready to unlock the new navigation + layout system and other V2 features

Read the migration guide ( https://scalar.com/products/docs/configuration/migration )

Migration tip: run npx @scalar/cli project upgrade and npx @scalar/cli project preview before publishing.

Questions or issues?

If you have any questions about the automatic upgrade, or want help with config migration, reply to this email or contact us at support@scalar.com ( support@scalar.com ).
