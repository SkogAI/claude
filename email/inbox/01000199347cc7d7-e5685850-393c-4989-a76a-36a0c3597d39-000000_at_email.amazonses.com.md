Hi Emil Skogsund,
We’ve taken JS/TypeScript tracing up a notch: The Langfuse JS/TypeScript SDK v4
[https://langfuse.com/docs/observability/sdk/typescript/overview] is now out of beta!
We’ve rebuilt this SDK on top of OpenTelemetry to improve the developer experience, make context management more robust, and allow
for easy integrations with the JS/TS ecosystem (Provider SDKs, Vercel AI SDK, LangChain JS, Mastra, ...).


Why you’ll love v4

 * First-class OpenTelemetry [https://opentelemetry.io/] support with robust context propagation, thus interoperable with all
   OpenTelemetry-based tools and external instrumentation libraries

 * New DX for tracing that automatically nests all observations without passing IDs around: Context managers, decorators, and
   manual control

 * Modular packages: @langfuse/core, @langfuse/client, @langfuse/tracing, @langfuse/otel

 * Autoinstrumentation for OpenAI SDK, LangChain JS, the Vercel AI SDK, and all OTel native external instrumentation libraries

 * Improved prompts, datasets, and scoring APIs

 
→ See the documentation [https://langfuse.com/docs/observability/sdk/typescript/overview]
→ Upgrade now from v3 to v4 [https://langfuse.com/docs/observability/sdk/typescript/upgrade-path]
