Hey there,
 
We are sending a clarification following our last email sent on 25 June regarding deprecation of v1 API endpoint. We heard two
major queries:
 
 1. Is the whole v1 API being deprecated?

 1. No, just a single POST endpoint of v1 API (POST /api/v1/integrations) is being sunset.

 1. June 30 is too soon, can we get more time?

 1. Absolutely. The new date for deprecation is August 1 2025. Hope you can make the necessary changes smoothly over this period.

 


👇🏼 Recapping everything again

 


🛠️ What’s Actually Changing

 * Only the following v1 POST endpoint will be deprecated:

 * POST /api/v1/integrations

 * All other v1 endpoints (GET, PUT, DELETE, and other POSTs) will remain fully supported.

 


🔄 Next Steps

 1. Update your calls to use POST /api/v3/auth_configs.

 1. Switch to nano IDs instead of UUIDs in responses (see ID Migration Guide
    [https://v3composio.docs.buildwithfern.com/api-reference/migration/get-migration-get-nanoid]).

 1. Rename “integrations” → “auth_configs” in your code and documentation.

 


📅 New Timeline

 * From now to July 31, 2025: Both v1 POST integrations and v3 auth_configs endpoints available

 * August 1, 2025: v1 POST integrations endpoint discontinued
