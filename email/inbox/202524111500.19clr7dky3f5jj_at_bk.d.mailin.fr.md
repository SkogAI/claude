Hi Dify Builders, 

 

We just shipped **Workflow Trigger** in [Dify v1.10.0](https://b6usd.r.a.d.sendibm1.com/mk/cl/f/sh/7nVU1aA2nfsTSeY62oltNKjy2JTL2Wc/C6I35zZl9ATH). This feature allows your workflows to listen for events and run automatically in the background.

**TL;DR:**
==========

- **Workflows can now stay online and react to events as background services.**

- 

**Three trigger types: Schedule, Plugin, and Webhook.**

- **Support added for the latest 5.1 series and Gemini 3 series models.**

**Why Trigger**
===============

Start logic often ends up scattered across buttons, cron jobs, and ad hoc scripts. This makes it hard to debug or track what actually initiates a flow.


Trigger brings this logic back to the Workflow canvas. You define when a workflow should start right alongside what it should do. 


The Start node supports two primary modes: **User Input** for on-demand runs, and **Trigger** for continuous event listening. You can even configure multiple triggers on the same canvas to handle different scenarios.

**Three ways to trigger**
=========================

- **Schedule Trigger:** A visual cron editor. Perfect for daily reports, recurring maintenance, or periodic checks. 
- **Plugin Trigger:** Native integration with tools like Slack, GitHub, and Linear. We handle the OAuth and parsing so you just handle the logic. 
- **Webhook Trigger:** Catch payloads from anywhere. Use it to connect internal back office apps, IoT devices, or custom scripts.

**Also in this release**
========================

- **New Models:** Support added for GPT-5.1 and Gemini 3 series models. Plug them directly into workflows to use their newer reasoning capabilities.

**See it in action: Automated Code Reviews**
============================================

Instead of writing glue code, you can now visually chain steps. For example, use a Plugin Trigger to detect a new PR on GitHub, have an LLM summarize the code changes, and automatically post the report to your Slack channel. [Read the blog](https://b6usd.r.a.d.sendibm1.com/mk/cl/f/sh/7nVU1aA2nfuMSB3P1O0LPh3mCfy7mT8/qXW9qMTbYnz4).

You can find more details in the [documentation](https://b6usd.r.a.d.sendibm1.com/mk/cl/f/sh/7nVU1aA2nfwFRhYhzxEnS3NaN2SuWPe/JpTkXJhQVb22).
