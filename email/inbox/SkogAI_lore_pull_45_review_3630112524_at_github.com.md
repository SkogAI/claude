## Pull request overview

This PR adds a file-based queue system for offline and batch lore generation, enabling users to queue tasks during work sessions and process them later with flexible LLM provider support. The implementation follows a state-machine architecture with priority-based FIFO processing.

**Key changes:**
- File-based queue with state transitions: `pending/` → `processing/` → `completed/`|`failed/`
- CLI tools for task management (`queue-task.sh`) and batch processing (`process-queue.sh`)
- Priority levels (high/normal/low) with persona association support
- YAML prompt templates for reusable generation patterns across manual and queued workflows
