**🌊 Waves 8 & 9: Full Power Unleashed**
=======================================

 

**Hello Surfers 👋**

 

We know we’ve rolled out some big updates in the past, but trust us when we say, this is a *big one* . 

**Wave 8** gave you total control over how Cascade behaves across projects, teams, and workflows.
 

Now, **Wave 9** delivers our first in-house model family: **SWE-1**, built not just to code – but to engineer.

 

Whether you’re working solo, leading a team, or managing an enterprise rollout, this double wave is for you. Let’s get into it.


**Meet the SWE-1 Family**
-------------------------


Three models. One goal: accelerate real software engineering.


- **SWE-1**: Our tool-capable, high-reasoning frontier model, now available for unlimited use to all paid users. 
- **SWE-1-lite**: Replacing Cascade Base, for an improved foundation, better, faster, and **free** for everyone. 
- **SWE-1-mini**: Optimized the current Windsurf Tab mode, bringing a fast, lightweight passive coding experience. Now **available unlimitedly**  to all users! 

###**Why SWE-1?**
Coding is just one slice of engineering. SWE-1’s model family spans editors, terminals, planning, feedback loops, and messy long-lived projects, designed for reasoning across the **entire lifecycle.**
 


###**Proven in Production**
We’ve tested SWE-1 in real-world repos and seen performance on par with top foundation models, but at lower cost and faster speeds.

 


**Custom Workflows**
--------------------

Repeat a common set of steps with minor tweaks? Turn it into a saved, shareable workflow.
 


- Create .md files in .windsurf/workflows/ 
- Define repeatable step-by-step instructions 
- Invoke them with a simple /command 
These live in your repo and are instantly shared with teammates—turning best practices into reusable automation.


**File Based Rules (Revamped)**
-------------------------------


The original .windsurfrules is evolving. Now you can define **multiple Rule files** in .windsurf/rules/  — each with its own scope.
 


- Use **Manual**, **Always On**, **Model Decision**, or **Glob** as activation modes 
- Easily reference Rules with @mentions inside Cascade 
- Maintain **Global Rules** that apply beyond any specific repo 
 

Customize how Cascade behaves per-project, per-team, or org-wide.


**🧠 Simultaneous Cascades**
---------------------------


No more waiting around while Cascade runs. Now you can start a new Cascade conversation even if one is already executing.
 


- Jump between Cascades freely 
- Get notified when action is needed in the background thread 
- Great for reviewing work, running heavy models, or answering quick side questions 
 

And we’ve done some behind-the-scenes magic too. Using smart checkpointing and summarization, Cascade now performs better in **long conversations**—so response quality holds steady even as chats get complex.

 

Power users just leveled up. 👀


**Casacde Plugins Panel**

Managing MCP tools just got way easier.


- Discover, add, and manage MCP servers 
- Enable/disable individual tools 
- See what’s live and what’s not—all from a single dashboard 
 

We’ve made MCP integration more transparent and more usable.

 

Control everything from your [admin portal](https://windsurf.com/team/windsurf_settings).

We’ve extended our support for the Model Context Protocol:
