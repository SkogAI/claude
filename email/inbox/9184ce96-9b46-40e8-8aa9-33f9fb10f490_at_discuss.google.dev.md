    [Appsheet feels slugish recently][3]

      Thanks for raising the issue. We confirmed an increased latency in editor load time starting yesterday and we believe we have addressed the underlying issue - load time appears to be returning to the prior baseline. As far as app sync time, peak latency still looks comparable to what we routinely saw before the recent holidays. There is a known issue affecting latency of data type conversion as discussed here, a fix for that should be coming this week. 
Regarding differences in sync time reported in the performance analyzer vs what was perceived by the user, the main reason is likely that the reported sync time only reflects the time spent by the server on the request, and not the time required for the client to download all of the app data over the network. So things like network speed and bandwidth can affect this, and the effect is larger for apps with more data.

    [Limit decimal points of percentage columns in chart view][5]

      If you use the sheet solely as a datasource, it is highly recommended that ALL columns be defined as “General Text”.  This prevents several potential datatype issues. 
If you must define a data type in the sheet because it is used elsewhere, keep in mind that AppSheet stores the raw value as Decimal for a Percent column in AppSheet.  You would then want the sheet column defined as “Decimal” with 2 decimal places to match your configuration in the app. 
Make the above changes and then force your “bad” value to be recalculated and stored again.  That should rectify the extra decimal positions. 
Any other columns that do not need to be type defined for external use, define them explicitly as “General Text” (not automatic)!!! 
I hope this helps!
