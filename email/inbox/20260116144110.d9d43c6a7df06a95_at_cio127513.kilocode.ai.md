We just launched Kilo for Slack, which allows you to spin up a cloud agent in Slack. You can read more about it on our blog ( https://blog.kilo.ai/p/announcing-kilo-for-slack ) and VentureBeat. ( https://venturebeat.com/technology/kilo-launches-ai-powered-slack-bot-that-ships-code-from-a-chat-message )

Here’s what you can do with Kilo for Slack:

- Ask questions about your codebase:

@Kilo how is error handling implemented in the payment module?

- Debug issues on the fly:

@Kilo I'm seeing this error in production: [paste your stack trace]. What's causing it? Can you push a PR to fix it?

- Implement fixes from Slack discussions:

@Kilo can you implement the caching improvements we discussed in this thread?

- Push PRs without leaving Slack:

@Kilo can you change "2025" to "2026" through all of the files in our kilo-org/kilocode repo?

The bot reads the full Slack thread, accesses your connected GitHub repos, and either answers your question or creates a branch and PR directly.

How to get started for free:

1. Connect your GitHub repos at app.kilo.ai → Integrations ( https://app.kilo.ai/integrations )

2. Add the Slack integration from the same page. ( https://app.kilo.ai/integrations )

Choose from a paid model (the bot will use your existing Kilo balance) or a free model (MiniMax 2.1 is free to use on Kilo for Slack for the next 7 days).

3. Mention @Kilo in any channel or DM.
