Langfuse now includes an OpenTelemetry backend to receive OTel traces at: /api/public/otel
> What is OpenTelemetry?
> OpenTelemetry [https://opentelemetry.io/] is an open-source observability framework for collecting and processing telemetry
> data. It includes Semantic Conventions, which standardize the naming and structuring of trace data.

WeвҖҷre thrilled about this update because it enables integrations with a wide range of frameworks and languages that use the
OpenTelemetry standard. This includes:
 * CrewAI [https://langfuse.com/docs/integrations/crewai]

 * AutoGen [https://langfuse.com/docs/integrations/autogen]

 * Semantic Kernel [https://langfuse.com/docs/integrations/semantic-kernel]

 * Pydantic AI [https://langfuse.com/docs/integrations/pydantic-ai]

 * Spring AI [https://langfuse.com/docs/integrations/spring-ai]

 * Smolagents [https://langfuse.com/docs/integrations/smolagents]

 * Other languages such as Java, Go, .NET

вҶ’ Learn more in our documentation [https://langfuse.com/docs/opentelemetry/get-started]

вҡЎ Open Source LLMOps Stack

В 



В 
We've partnered with LiteLLM [https://www.litellm.ai/] to introduce the Open Source LLMOps Stack [https://oss-llmops-stack.com/].
В 
Selecting the right tooling stack is essentialвҖ”avoiding vendor lock-in and high switching costs means you can continue to develop
in rapid cycles.
В 
Popular among Langfuse and LiteLLM users (especially in enterprise and regulated settings), this stack enables efficient
observability without extensive instrumentation. LiteLLM further enhances the experience with virtual LLM keys, budgeting,
failovers, and compatibility with over 100 LLMs through the OpenAI API schema.
вҶ’ Learn more: oss-llmops-stack.com [https://oss-llmops-stack.com/]
