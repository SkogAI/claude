QoL improvements
 
We have made several improvements to the quality of life of our users with a slew of small but important fixes:
 
 * In the sidebar, when creating folder/prompt, save it when typing the name and unfocusing the input

 * If there’s only one folder in the project, display it open

 * When creating a prompt put focus in the editor

 * When opening evaluations, close parameters

 * Collapsed/Expanded parameters are now displayed properly

 * Redesigned run prompt, run experiment, add message, reset buttons in playground

 * Renamed “Publish prompt” to “Share” in playground

 * Moved toasts to the left side to avoid overlaps

 
Other fixes
 
 * Shaved several GBs off the public Docker images

 * Added several new MCPs to the list of supported integrations.

 * Improved parameter mapping by using selected dataset columns as a fallback.

 * Refactored jobs to remove meta programming, potentially fixing a memory leak.

 * Addressed a build issue with heapdump in amd64 environments.

 * Improved SDK’s error handling to address silent failures.
