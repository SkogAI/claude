Agno updates




Agno v2.5: Filling in the production layer

This release fills in the connective tissue between "I built an agent" and "we run agents as a business capability."
 
Look at what shipped not as a feature list, but as a set of bets about what production agent systems actually need:
 
 * Teams execution modes: Teams now support four distinct execution modes: coordinate, route, broadcast, and tasks, giving you
   explicit control over how agents collaborate.

 * Learning Machines for Teams: Teams can now accumulate and persist knowledge across sessions and runs, retaining context about
   goals, constraints, and past outcomes without manual re-prompting.

 * Approval system: Require explicit human authorization before agents execute sensitive actions. Add blocking or audit-mode gates
   with persistent records and a dedicated Approvals API so high-stakes operations never proceed without sign-off and every
   decision leaves an audit trail.

 * Built-in scheduler: You can now create and manage cron schedules for agents, teams, and workflows directly through the
   Scheduler SDK and AgentOS. Define recurring jobs with support for retries, timeouts, and timezone configuration, with no
   external infrastructure required.

 * Richer HITL for teams: The HITL system for teams now supports tool confirmation, user input collection, and external tool
   execution, so multi-agent workflows can pause at exactly the right moment to get human approval, gather additional context, or
   wait for an external action to complete.

 
The framework gets you started. AgentOS gets you deployed. This completes the picture. See what shipped →
