See the Responses API reference → ( https://portkey.ai/docs/api-reference/inference-api/responses/responses?cr_email=ZW1pbC5za29nc3VuZEBnbWFpbC5jb20%3D&utm_campaign=Jan%2726+updates&utm_content=Jan%2726Updates&utm_medium=email_action&utm_source=customer.io )

Unified Rerank API

We've added a single /rerank endpoint so you can switch between Cohere, Jina, and Voyage AI without touching your app – same interface, pick the provider that fits.

GUARDRAILS

Azure Shield Prompt & Protected Material

Azure Shield Prompt and Azure Protected material detection

Azure Shield Prompt – Helps you catch jailbreak and prompt injection by scanning system prompts and user messages; use an API key or Entra ID to connect.

Azure Protected Material – Scans LLM outputs for known copyrighted or protected text so you can stay on the right side of IP requirements.

Add Azure guardrails → ( https://portkey.ai/docs/integrations/guardrails/azure-guardrails?cr_email=ZW1pbC5za29nc3VuZEBnbWFpbC5jb20%3D&utm_campaign=Jan%2726+updates&utm_content=Jan%2726Updates&utm_medium=email_action&utm_source=customer.io#azure-shield-prompt )

Sequential Guardrails Execution

You can now run guardrail checks one after another with the sequential flag – handy when later checks depend on earlier ones or when order matters for compliance.

Set up sequential guardrails → ( https://portkey.ai/docs/product/guardrails?cr_email=ZW1pbC5za29nc3VuZEBnbWFpbC5jb20%3D&utm_campaign=Jan%2726+updates&utm_content=Jan%2726Updates&utm_medium=email_action&utm_source=customer.io#there-are-7-types-of-guardrail-actions )
