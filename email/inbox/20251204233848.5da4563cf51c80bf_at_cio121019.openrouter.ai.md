Hi there,

Today, we’re launching Broadcast ( https://openrouter.ai/docs/guides/features/broadcast/overview?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io ) (beta): a simple way to automatically send traces from your OpenRouter requests to external observability platforms with no additional code.

Give product, data science, and engineering teams access to trace data directly within the tools they already use

Why “Broadcast”:

* Monitoring: Monitor and debug production issues faster (errors, latency, tool calls, and more).
* Analytics: Track and analyze usage and cost over time (by model, provider, app, or user).
* Simplicity: Integrate with
your existing observability workflows with no extra work or latency.
* Filtering: Optionally filter traces by API key.
* Sampling: Define sample rates for each destination.

Supported Destinations: We’re launching six integrations, with more on the way.

* Braintrust ( https://openrouter.ai/docs/guides/features/broadcast/braintrust?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io )
* Langfuse ( https://openrouter.ai/docs/guides/features/broadcast/langfuse?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io )
* Weights & Biases - CoreWeave ( https://openrouter.ai/docs/guides/features/broadcast/weave?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io )
* Datadog ( https://openrouter.ai/docs/guides/features/broadcast/datadog?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io )
* OpenTelemetry Collector ( https://openrouter.ai/docs/guides/features/broadcast/overview?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io )

How to use Broadcast: Visit your settings ( https://openrouter.ai/settings/broadcast?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io ) page and connect to any platform.

Coming Soon: We are continually adding new features to Broadcast.

* Data Governance: Remove PII and GDPR data before sending traces to destinations.
* Advanced filtering: Cut costs by only sending specific traces you need to each platform.
* Replay: Recent traces to different destinations.
* Exports: Export trace data to JSONL, Parquet, or CSV.
* Destinations: 25+ more destinations.

Pro tip: You can associate traces with specific users or sessions in your application. Learn more here ( https://openrouter.ai/docs/guides/features/broadcast/overview?utm_campaign=Broadcast-Beta-Release&utm_content=Broadcast-Beta-Release&utm_medium=email_action&utm_source=customer.io#optional-trace-data ).
