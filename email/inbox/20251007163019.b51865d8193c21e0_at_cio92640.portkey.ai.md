Unified token counting endpoint

Unified Token Counting Endpoint

Token counting is now unified across AWS Bedrock, Vertex AI, and Anthropic.

With a single endpoint, you can:
* Estimate token usage before sending a request (avoid context limit errors)
* Standardize cost/usage logic across providers
* Enforce routing and quota rules directly inside your app

GUARDRAILS

Metadata-based Model Access guardrail

Metadata-based Model Access Guardrail

We’ve added a new guardrail that lets you control model access based on metadata key-value pairs. It checks metadata at runtime for every request, so you get fine-grained governance with zero changes to your app.

Regex-replace guardrail

Regex Replace Guardrail

Go beyond standard PII protection. Define custom regex patterns, mask matches (e.g., [masked_user]), and enforce redaction at runtime. Ideal for internal IDs or project codes. Read more here ( https://portkey.ai/docs/integrations/guardrails/regex?cr_email=ZW1pbC5za29nc3VuZEBnbWFpbC5jb20%3D&utm_campaign=Sept%2725+updates&utm_content=Sept%27+25+Updates&utm_medium=email_action&utm_source=customer.io )

GATEWAY & PROVIDERS

Unified finish_reason parameter

We’ve standardized the finish_reason field across all providers. By default, values now follow OpenAI-compatible outputs for consistent behavior in multi-provider setups.
To keep original provider values, set
x-portkey-strict-openai-compliance = false.

Conditional router enhancement

Conditional routing now supports parameter-based routing in addition to metadata. Parameter-based routing enables dynamic, per-request optimizations, giving you better performance, cost efficiency, and control over user experience. Read more here ( https://portkey.ai/docs/product/ai-gateway/conditional-routing?cr_email=ZW1pbC5za29nc3VuZEBnbWFpbC5jb20%3D&utm_campaign=Sept%2725+updates&utm_content=Sept%27+25+Updates&utm_medium=email_action&utm_source=customer.io )

* Added support for task_type and dimensions parameters in Vertex AI batch embeddings
* Support for global region
* OpenAI & Azure OpenAI:
* Added new parameters for GPT-5 compatibility

* AWS Bedrock:
* Added video support in chat completions
* Added support for APAC cross region inference profiles
* Added support for performance_config parameter
* Azure Foundry → Updated the parameter mapping to support all the latest OpenAI compatible chat completions parameters. You can now also enable multiple models in a single integration
* Adding new models to Azure OpenAI → Now much simpler with direct model fetching from target URI

General improvements:

* Resolved Authorization header conflict for passthrough requests
* Custom Header support for OTEL Export of analytics data
