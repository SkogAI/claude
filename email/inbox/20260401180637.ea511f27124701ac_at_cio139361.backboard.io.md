Hey SkogAI,

Maya here. I run DevRel at Backboard.

When people start building agents with stateless APIs, the first demo is easy, and then the same questions show up fast: where does the conversation state live, how do you keep memory consistent across sessions, and how do you keep tool calling sane once you have more than one workflow or model.

That’s the problem we’re working on with Backboard.

If you want to try it, you can create a free Backboard account here ( https://app.backboard.io/signup ). It comes with dev credits by default, and free state management for life. If you hit a ceiling while testing a real use case, reply and I’ll top up credits.

Quick overview, in plain English: Backboard is the layer that handles threads and session state, persistent memory, tool calling plus RAG, and routes to thousands of models, or you can BYOK from OpenRouter, OpenAI, Anthropic etc.

If you prefer to see it once before touching anything, Jasmine recorded a quick getting started video here that we use for hackathons. https://www.youtube.com/watch?v=_9z105gqLqE

I’m curious: what are you using today for state and memory, and is it something you want to manage long term or was it just because you needed something?
