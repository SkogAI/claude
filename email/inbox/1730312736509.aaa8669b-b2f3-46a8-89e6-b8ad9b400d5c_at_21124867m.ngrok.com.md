## Changelog

🚀 2024-10-24: Pushed a new ngrok agent to fix a bug where 4-character timezones weren't handled properly in the agent TUI. Thanks to @danielcompton for reporting this in our community repo! (v3.18.1)

🚀 2024-10-17: Released the ngrok agent with a new --name field, which lets you specify the endpoint's name as it appears in the dashboard, logs, API calls, and more. (v3.18.0)

🚀 2024-10-15: Added a JSON/YAML toggle to the in-dashboard Traffic Policy editor so that you can switch between them without closing the editor first.

🚀 2024-10-14: 0 and 1 TCP addresses no longer share IPs with HTTP endpoints to reduce confusion and improve IP reputation.

🚀 2024-10-10: Released the ngrok agent with new endpoints create/update/delete subcommands to the ngrok api command, and adds support for policy phases like on_tcp_connect, on_http_request, and on_http_response. We also fixed a bug where agent endpoints incorrectly used TLS instead of HTTPS—thanks to @franck102 for identifying the issue! (v3.17.0)

🚀 2024-10-09: Loading the Edges page in the dashboard is now significantly faster when your account contains 10 or more edges.

🚀 2024-10-03: Released the ngrok agent for Windows on ARM64. (v3.16.1)

🚀 2024-09-26: Updated the in-dashboard examples of agent configurations to the new v3 format using endpoints.
