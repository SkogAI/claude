We're excited to introduce our Batch API–an asynchronous endpoint built to 
efficiently handle large volumes of requests. It streamlines workflows by 
removing the complexities of managing many synchronous requests. This makes it 
ideal for tasks that don’t require real-time responses, such as embedding large 
corpora or running evaluations.

Read the blog  <https://mdb.link/voyageai-batchapi-blog>



Compared to our synchronous API, the Batch API:

 * Simplifies large-scale workflows by eliminating the need to manage queues, 
retries, threading, or rate limits
 * Offers higher throughput and limits, supporting up to 1GB file sizes (5x 
OpenAI’s 200MB), 100k inputs per batch (2x OpenAI’s 50k limit), and 1B tokens 
per organization
 * Delivers cost savings of 33%, making it a cost-effective option at scale
Our Batch API offers a 12-hour completion window—half of market alternatives, 
including OpenAI. This is great for overnight runs and allows you to maximize 
daytime productivity.



Learn more <https://mdb.link/voyageai-batchapi-docs> about our Batch API and 
try it today.
