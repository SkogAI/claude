It’s been a wild start to the year. Here’s a summary of our product updates from January!

Introducing Generate

 

With Generate, you can ask how to do a task within a long-running or interactive CLI command. For example:

* Database REPL (psql, mysql, sqlite): “Create a table to store user data”
* Text editors (vim, nano): “How do I quit vim?”
* Python REPL (ipython, python): “Create a simple plot of data.csv”
* Cloud provider shells (gcloud, aws cli): “Create a new Kubernetes cluster”

Read more about Generate in our
documentation ( https://docs.warp.dev/features/warp-ai/generate#id-2.-generate-text-and-contextual-suggestions-in-interactive-clis )
AI Improvements

*
We launched support for DeepSeek R1 and V3 ( https://x.com/warpdotdev/status/1885376165910896827 ) and o3-mini ( https://x.com/warpdotdev/status/1885470031871697160 ) in Agent Mode.
* We launched autonomous mode ( https://x.com/warpdotdev/status/1886288727095546217 ), which allows you to auto-execute read-only commands without having to constantly press enter.
Core Features and Fixes

*
We are working on over 20 bugs & quality-of-life fixes in Warp, including custom cursor types ( https://x.com/warpdotdev/status/1876692210365984873 ), supporting marked text ( https://x.com/warpdotdev/status/1884305615834722507 ), rendering more native box characters ( https://x.com/warpdotdev/status/1884305614408671415 ), and rectangular selection within a block ( https://x.com/warpdotdev/status/1886909757329641480 ).
* You can now define an alias for a workflow in your Warp Drive, as well as specify aliases with specific parameters. Watch a demo here ( https://x.com/warpdotdev/status/1886908952635695558 ).
*
Dynamic enums allows you to define a parameter for your workflow based on a shell command, instead of plain text. Watch a demo here ( https://x.com/warpdotdev/status/1886907414492144112 ).
