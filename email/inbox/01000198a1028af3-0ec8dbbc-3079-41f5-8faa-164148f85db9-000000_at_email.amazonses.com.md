The biggest news has been the release of GPT-5. The reports from others has been a mixed bag. Some people think it sucks, and others think it's the best model ever released.

My early thoughts on how it compares to Claude 4:

- I don't like the way Sam and OpenAI hype their products
- We're getting close now to a plateau in capabilities with these models. You could use Gemini 2.5, Claude 4, Grok 4, GPT-5 or open models like Qwen3-Coder and Kimi K2 and get great results with any of them.
- TLDR; use the model and tools that work best for you and your workflows
- It does seem very good for frontend UI that doesn't "look like AI" compared to Claude 
- The price is very cheap and makes it a great complement to existing subs like Claude Max (but it does use a lot of tokens for thinking)
- It actually seems on a par with Opus and better than Sonnet for working on bugs or refactors to existing code (vs building from scratch)
- Context engineering with this model is VERY important and you need to be more verbose, specific, and direct with your prompts. "didn't work, fix it" will not cut the mustard!

I want to test using Opus in Claude Code as the planner and GPT-5 as the pair programmer on some more complex apps and problems to really compare.
