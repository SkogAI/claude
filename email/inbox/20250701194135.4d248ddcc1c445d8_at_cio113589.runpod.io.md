Our S3 API is now live in beta ( https://www.runpod.io/console/user/storage )

************************************
Announcing our new S3-Compatible API
************************************

Runpod's S3-compatible API ( http://docs.runpod.io/serverless/storage/s3-api ) is officially live and production ready — giving you seamless, pod-free access to your network volumes.

If you’ve ever launched a pod just to move files or stitched together brittle scripts for data workflows, this changes the game.

*****************************
With the new S3 API, you can:
*****************************

*
Upload and retrieve files without spinning up compute
* Use standard tools like AWS S3 CLI and Boto3 with zero config hassle
* Integrate Runpod storage into any AI pipeline — no rewrites, no friction
* Manage data with clean, object-level control for teams, models, and apps

From training LLMs to serving inference at scale, this gives you faster, simpler, and more reproducible workflows — making Runpod a truly unified AI backend.

Read the docs here ( http://docs.runpod.io/serverless/storage/s3-api ).

****************************
Here’s how to get started:
****************************

*
Create a Network Volume - Head to the Storage ( https://console.runpod.io/user/storage ) tab in your Runpod console and select or create a network volume.
* Connect with Any S3 Tool - Point your tool of choice (e.g., AWS S3, Boto3) at your datacenter endpoint and plug in your access keys.
* Start Moving Data - Upload datasets, stream into pods, or store results — all via S3-native commands.
