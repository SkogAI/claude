Warp 2.0 - The First Agentic Development Environment

 

( https://www.youtube.com/watch?v=k9BRhSbVSpI )
Today, we're excited to launch Warp 2.0, the Agentic Development Environment. Warp 2.0 is an evolution from an intelligent terminal to an all-in-one workbench built to support full-lifecycle development with agents.

In Warp 2.0, you get:

* The top overall coding agent: #1 on Terminal-Bench and 71% on SWE-bench Verified
* Agent multi-threading and management with the ability to run and monitor multiple agents – all under your control
* The best interface for agentic development: from our Universal Input to our inline code editor – Warp 2.0 is the first workbench designed for developing by prompt

----------------------
What's new in Warp 2.0
----------------------

State-of-the-art coding agent, with a built-in code editor

Warp’s coding agent scored 71% on SWE-bench Verified and ranks #1 on Terminal-Bench (52%) - making it the best overall coding agent in the world. Prompt the agent, watch it gather context and create a plan; as the agent runs, it will loop you in if it needs help – and you can make edits directly in our inline code editor.

Multi-thread yourself with agents

Run and manage multiple agents natively within Warp. Prompt agents, attach context, and watch them build features, fix bugs, and resolve crashing servers. Grant them full autonomy or limit what they can do – you have full control of their actions. Track progress in Warp's agent management pane.

Still the best command-line

Warp still has the best terminal UX on the market. All the innovations that make Warp powerful – our rich editor for commands, our block based output – work as they always have, and are now usable in guiding agents, not just writing commands.

Shared context for humans and agents

By virtue of Warp's low layer in the stack and its built-in knowledge base, Warp Drive, it's easy for your teammates – AI and human – to do any task. In addition, Warp's agents have access to MCP, rules, and anything that can be gathered by running a CLI, which is pretty much anything.
